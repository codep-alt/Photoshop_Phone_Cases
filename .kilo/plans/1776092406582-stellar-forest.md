# Plan: Add Master File Support to Generate Images

## Goal
Replace the current title-parsing logic with a Master File lookup approach. Match `SKU + NL_Variant` from the import file to the Master file to get phone model, phone brand, and case brand.

## Data Structures

### Master File (`MASTER FILE ON CSV.csv`)
| Column | Example | Purpose |
|--------|---------|---------|
| SKU | CASIM0225 | Match key |
| NL_Category_1 | Samsung | Phone brand |
| NL_Category_2 | Samsung Galaxy A23 | Phone model/title |
| NL_Variant | Solid_Beige, Solid_Brown, Solid_Black | Match key + color |
| Brand | Casimoda | Case brand |

### Import File (`Samsung A23 import - Casimoda.xlsx - Blad1.csv`)
| Column | Example | Purpose |
|--------|---------|---------|
| Internal_ID | 1 | Order ID |
| SKU | CASIM0225 | Match key |
| NL_Variant | Solid_Beige | Match key |
| NL_Title_Long | "Samsung Galaxy A23 beige case - Marmer waves" | Extract design name |
| NL_Category_1 | Samsung | (from master, not import) |
| NL_Category_2 | Samsung Galaxy A23 hoesjes | (from master, not import) |

## Implementation Steps

### 1. Add Master File Loading (`generateImagesAuditor.ts`)

Create `MasterRow` interface:
```typescript
interface MasterRow {
  sku: string;
  nlCategory1: string;  // Phone brand
  nlCategory2: string;  // Phone model
  nlVariant: string;    // Match key
  brand: string;         // Case brand
}
```

Add function `loadMasterFile(masterPath: string): Map<string, MasterRow>`
- Reads the master Excel/CSV file
- Creates a `Map` with key `${SKU}_${NL_Variant}` for fast lookup
- Returns the map

### 2. Modify `auditGenerateImagesOrders()` Signature

Add new parameter `masterPath: string`:
```typescript
export async function auditGenerateImagesOrders(
  csvPath: string,
  masterPath: string,  // NEW: path to master file
  mockupsBase: string,
  designsBase: string
): Promise<GenerateImagesResult>
```

### 3. Update Matching Logic in Audit Loop

**OLD flow:**
1. Read `NL_Title_Long` from import
2. Parse title to extract model, color, design, variant
3. Build mockup path using extracted model/color

**NEW flow:**
1. Read `SKU` and `NL_Variant` from import
2. Lookup master file with key `${SKU}_${NL_Variant}`
3. If not found → skip with reason "Not found in master file"
4. If found → get `NL_Category_2` (phone model), `NL_Category_1` (phone brand), `Brand` (case brand)
5. Extract design name from `NL_Title_Long` (using existing `parseTitle` logic)
6. Apply color mapping (zwart→Black, bruin→Brown, etc.) to `NL_Variant` for folder path

### 4. Color Mapping for NL_Variant

The `NL_Variant` in master file uses English colors (Black, Brown, Beige), but import may use Dutch (Zwart, Bruin). Apply existing colorMap to extract color before matching:

```typescript
// Extract color from NL_Variant (e.g., "Solid_Bruin" → "Bruin" → "Brown")
const colorPart = nlVariant.split("_")[1]?.toLowerCase() || "";
const colorEnglish = colorMap[colorPart] || colorPart;
// Lookup key becomes "CASIM0225_Solid_Brown"
```

### 5. Update UI (`main.tsx`)

Add new file picker for "Master File":
```tsx
<PickerItem
  label="Master File"
  value={masterPath}
  onPick={() => handlePickFile("Master File", setMasterPath, "genImagesMasterPath")}
/>
```

Update `handleTest` and `handleGenerate` to pass `masterPath` to `auditGenerateImagesOrders`.

Keep existing title-parsing path as fallback OR require master file when the feature is enabled. The user can choose between:
- **Master Mode**: Uses master file for matching
- **Legacy Mode**: Uses title parsing (existing behavior)

For simplicity, initially require master file when this feature is used.

## Skip Reasons (for stats)

When a row doesn't match:
- `"Not found in master file"` - SKU+NL_Variant combination not in master
- `"Empty SKU"` - No SKU in import row
- Existing reasons (mockup not found, design not found, etc.)
